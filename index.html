How The Rule Engine Works

Deterministic logic steps convert selected symptoms into explainable and auditable diagnostic suggestions.

01. SYMPTOM INTAKE

Input Validation

User selects 2 to 6 valid symptoms. Unsupported inputs are rejected before diagnosis.

GUARDRAIL ENABLED
02. RULE MATCHING

Deterministic Rules

System checks exact symptom combinations first, then tries closest partial rules when exact match is absent.

EXPLAINABLE LOGIC
03. SAFETY LAYER

Confidence & Warnings

Confidence is adjusted for unrelated symptoms and warnings are emitted to avoid blind trust in output.

RISK-AWARE OUTPUT
04. ACTIONABLE RESULT

Clinical Suggestion

Result includes disease name, matched rule, suggested tests, specialist, and optional missing symptom hints.
